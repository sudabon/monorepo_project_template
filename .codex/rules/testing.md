# テスト方針

## バックエンド

- ユニットテスト: domain, app 層を中心にテストする。リポジトリはモックする。
- 統合テスト: infra 層のDB操作等を実DBまたはテスト用DBで検証する。
- テストファイルは `tests/unit/`, `tests/integration/` に配置する。
- テスト関数名は `test_<対象>_<条件>_<期待結果>` の形式にする。
- フィクスチャは `conftest.py` に共通化する。

## フロントエンド

- ユニットテスト（Vitest）: カスタムフック、ユーティリティ関数、複雑なロジックを持つコンポーネントを対象にする。
- E2Eテスト（Playwright）: ユーザーフロー単位で `frontend/e2e/` に配置する。
- テストファイルは対象ファイルと同階層に `*.test.ts(x)` として配置する。
